# 📚 Online Bookstore SQL Analysis

## 📌 Objective
Analyzed an online bookstore database to uncover revenue trends, 
customer behavior, and inventory insights using MySQL.

## 📂 Dataset
3 interconnected tables with 500+ records:
| Table | Description |
|-------|-------------|
| Books | Book details — title, author, genre, price, stock |
| Customers | Customer info — name, email, city, country |
| Orders | Transaction data — order date, quantity, total amount |

## 🛠️ Tools Used
- MySQL
- MySQL Workbench

## 💡 Business Questions Solved

### Basic
- Retrieve all books by genre
- Find books published after 1950
- List customers from Canada
- Total stock and revenue calculation
- Find most/least expensive books

### Advanced
- Customers who placed at least 2 orders
- Most frequently ordered book
- Top 3 expensive Fantasy books
- Cities where customers spent over $30
- Stock remaining after fulfilling all orders

## 🧠 SQL Concepts Used
- JOINs (INNER JOIN)
- Aggregate Functions (SUM, COUNT, AVG, MAX)
- GROUP BY and HAVING
- Subqueries
- DISTINCT
- ORDER BY and LIMIT
- Aliases

## 🗂️ File Structure
| File | Description |
|------|-------------|
| bookstore_analysis.sql | All 20 queries (11 basic + 9 advanced) |
| Books.csv | Books dataset |
| Customers.csv | Customers dataset |
| Orders.csv | Orders dataset |
